## 三、后端详细设计

### 3.1 新增统一Agent接口

在 `agent` 包下创建 `UnifiedAgent.java`，替换原有的 `CodeReviewAgent` 接口，原有Agent需要实现新接口。

```java
package com.agent.codereview.agent;

public interface UnifiedAgent {
    /**
     * 执行任务
     * @param codeContent 代码（或需求描述）
     * @param language 编程语言
     * @param additionalContext 额外上下文（前端传入，可为空）
     * @return 执行结果（Markdown或代码文本）
     */
    String execute(String codeContent, String language, String additionalContext);
    
    String getAgentName();
    String getAgentDescription();
}
```

**重构要求**：将现有的 `LogicAgent`、`SecurityAgent`、`PerformanceAgent` 修改为实现 `UnifiedAgent` 接口，方法名由 `review()` 改为 `execute()`，同时修改 `MultiAgentReviewService` 中的调用。

### 3.2 新增Agent实现类

所有新Agent都放在 `com.agent.codereview.agent` 包下，使用与现有Agent相同的 `RestTemplate` 调用AI的模式。

#### 3.2.1 CodeGenerationAgent.java

```java
@Component
public class CodeGenerationAgent implements UnifiedAgent {
    @Value("${spring.ai.openai.api-key}") 
    private String apiKey;
    
    private final RestTemplate restTemplate = new RestTemplate();
    private final KnowledgeVectorService knowledgeVectorService;

    public CodeGenerationAgent(KnowledgeVectorService kvs) {
        this.knowledgeVectorService = kvs;
    }

    @Override
    public String execute(String description, String language, String additionalContext) {
        // RAG检索相似示例代码
        List<String> examples = knowledgeVectorService.searchSimilar(description + " " + language, 2);
        
        String systemPrompt = "你是一位资深" + language + "开发专家，请根据需求描述生成高质量的代码。要求代码可运行，包含必要的注释和异常处理。";
        String userPrompt = "需求描述：" + description + "\n";
        if (!examples.isEmpty()) {
            userPrompt += "\n参考示例代码：\n" + String.join("\n", examples) + "\n";
        }
        userPrompt += "\n请生成" + language + "代码：";
        
        return callApi(systemPrompt, userPrompt);
    }
    
    // getAgentName() 返回 "CodeGenerationAgent"， getAgentDescription() 返回一句描述
    // callApi 方法复用现有Agent中的调用逻辑（POST dashscope completions）
}
```

### 3.3 新增统一路由控制器

新建 `com.agent.codereview.controller.AssistantController.java`

```java
@RestController
@RequestMapping("/api/assistant")
public class AssistantController {

    private final Map<String, UnifiedAgent> agentMap = new HashMap<>();
    
    @Autowired
    public AssistantController(List<UnifiedAgent> agents) {
        for (UnifiedAgent agent : agents) {
            agentMap.put(agent.getAgentName(), agent);
        }
    }

    @PostMapping("/execute")
    public ResponseEntity<AssistantResponse> execute(@RequestBody @Valid AssistantRequest request) {
        UnifiedAgent agent = agentMap.get(request.getActionType());
        if (agent == null) {
            return ResponseEntity.badRequest().body(new AssistantResponse("error", "不支持的actionType"));
        }
        // 提交异步任务，返回taskId
        String taskId = UUID.randomUUID().toString();
        // 异步执行并存储结果（可复用现有Async机制，或新建一个AssistantTaskService）
        assistantTaskService.executeAsync(taskId, agent, request);
        return ResponseEntity.ok(new AssistantResponse("ok", taskId));
    }
}
```

**AssistantRequest DTO**：
```java
@Data
public class AssistantRequest {
    @NotBlank
    private String actionType;   // 对应 agentName，如 "CodeGenerationAgent"
    @NotBlank
    private String input;        // 代码或需求描述
    private String language;     // 默认 java
    private String extraContext; // 可选
}
```

**结果存储**：复用 `review_task` 表，但 `status` 和 `result_summary` 无需修改，或新增通用表 `assistant_task`。简单起见，直接复用现有表和轮询接口，需要修改 `ReviewController` 返回通用表述，或新增 `/api/assistant/task/{id}` 查询接口。

**推荐方案**：在 `review_task` 表增加 `task_type` 字段（VARCHAR），默认 `REVIEW`，新任务设定为对应的actionType。这样前端无需改动轮询逻辑，只需在历史记录中筛选项。

### 3.4 异步任务管理调整

修改 `ReviewTaskServiceImpl` 或新建 `AssistantTaskServiceImpl`，使其能根据 `task_type` 调用不同Agent的 `execute` 方法。异步方法逻辑：

```java
@Async
public void doAssistantTask(Long taskId) {
    ReviewTask task = reviewTaskMapper.selectById(taskId);
    task.setStatus(1); reviewTaskMapper.updateById(task);
    try {
        UnifiedAgent agent = agentFactory.getAgent(task.getTaskType());
        String result = agent.execute(task.getCodeContent(), task.getLang(), null);
        task.setResultSummary(result);
        task.setStatus(2);
    } catch (Exception e) {
        task.setStatus(3);
        task.setResultSummary("执行失败：" + e.getMessage());
    }
    reviewTaskMapper.updateById(task);
}
```

### 4.1 引入代码编辑器 Monaco Editor

**安装**：
```bash
npm install monaco-editor @monaco-editor/loader
```

**修改 `App.vue` 或新建组件 `CodeEditor.vue`**：
- 用 `monaco-editor` 替换现有的 `el-input` 文本域。
- 支持语言语法高亮、行号、差异对比（Diff Editor）。
- 提供 `editorRef` 给父组件用于获取/设置代码内容。

**示例代码片段**：
```vue
<template>
  <div ref="editorContainer" class="editor-container"></div>
</template>

<script setup>
import { ref, onMounted, watch } from 'vue'
import loader from '@monaco-editor/loader'

const props = defineProps({
  modelValue: String,
  language: { type: String, default: 'java' },
  readOnly: false
})
const emit = defineEmits(['update:modelValue'])

const editorContainer = ref(null)
let editor = null

onMounted(async () => {
  const monaco = await loader.init()
  editor = monaco.editor.create(editorContainer.value, {
    value: props.modelValue,
    language: props.language,
    readOnly: props.readOnly,
    theme: 'vs-dark',
    automaticLayout: true
  })
  editor.onDidChangeModelContent(() => {
    emit('update:modelValue', editor.getValue())
  })
})

watch(() => props.modelValue, (newVal) => {
  if (editor && editor.getValue() !== newVal) {
    editor.setValue(newVal)
  }
})
</script>
```
