# 智能代码审查助手 - 技术选型文档

## 1. 选型原则

- **稳定性优先**：选择成熟、社区活跃、长期维护的技术栈。
- **与项目匹配**：需满足异步处理、向量检索、AI 集成等核心需求。
- **降低复杂度**：优先使用 Spring 生态及常用工具，避免过度定制。
- **可扩展性**：支持后续增加多 Agent、消息队列、监控等模块。

## 7. 多 Agent 架构

| 组件 | 职责 | 状态 |
|------|------|------|
| **CodeReviewAgent（接口）** | 定义 Agent 规范，包含 `review()` 和 `getAgentName()` 方法。 | ✅ 已实现 |
| **LogicAgent** | 逻辑审查专家，检查空指针、边界条件、资源泄漏、异常处理等。 | ✅ 已实现 |
| **SecurityAgent** | 安全审查专家，检查 SQL 注入、XSS、硬编码密码、权限越权等。 | ✅ 已实现 |
| **PerformanceAgent** | 性能审查专家，检查时间复杂度、N+1 查询、缓存策略、内存使用等。 | ✅ 已实现 |

## 13. 关键技术说明

### 13.1 Spring AI 集成说明

**ChatClient（已成功集成）**：
- 使用 `ChatClient.Builder` 注入 `ChatClient` 实例
- 支持通过 `defaultSystem()` 设置默认系统提示词
- 调用方式：`chatClient.prompt().user(prompt).call().content()`

**EmbeddingClient（暂不可用）**：
- Spring AI 1.0.0-M4 版本的 `spring-ai-openai-spring-boot-starter` 中不包含 `EmbeddingClient` 类
- 当前使用 RestTemplate 作为临时方案，功能正常

### 13.2 多 Agent 架构设计

**设计思路**：
1. 定义 `CodeReviewAgent` 接口，统一 Agent 行为规范。
2. 三个专业 Agent（Logic、Security、Performance）各自专注于特定领域。
3. 使用 `CompletableFuture` 实现并行调用，提高审查效率。
4. `MultiAgentReviewService` 作为协调器，汇总三份意见生成最终报告。

### 13.3 双数据源配置

**实现方式**：
- MySQL 作为主数据源，存储任务元数据（使用 MyBatis-Plus）。
- PostgreSQL + pgvector 作为向量数据源，存储知识库向量（使用 JdbcTemplate）。
- 通过 `@Qualifier` 和 `@Primary` 注解区分数据源。

### 13.4 前端界面设计

**技术实现**：
- 使用 Vue 3 Composition API + Element Plus 构建。
- 渐变背景 + 毛玻璃效果头部导航。
- 深色主题代码编辑器区域。
- 实时轮询 + 加载动画。
- Markdown 报告美化渲染。
- 本地存储历史记录。
